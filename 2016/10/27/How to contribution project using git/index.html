How to contribution project using git

When you find some issues in project on Github, you want to fixed it, then submit a pull request, how you do this? Now these is docker contribution guide, I think that is good start for me and everyone who want to contribute for all other projects on GitHub.

Sync your fork and create a new branch

To sync your repository:

  1. Open a terminal on your local host.

  2. Change directory to the project root.

  3. Checkout the master branch

Recall that origin/master is a branch on your remote Github depository.

  1. Make sure you have the upstream remote owner/project by listing them.

If the upstream is missing, add it.

  1. Fetch all the changes from the upstream/master branch.

  2. Rebase your local master with the upstream/master.

This command applies all the commits from the upstream master to your local master.

  1. Check the status of your local branch.

Your local repository now has all the changes from upstream remote. You need to push the changes to your own remote fork which is origin master.

  1. Push the rebase master to origin master.

  2. Create a new feature branch to work on your issue. Your branch name should have the format XXXX-descriptive where XXXX is the issue number you are working on. For example:

Your branch should be up-to-date with the upstream/master.Why? Because you branched off a freshly synced master.

  1. Rebase your branch from upstream/master.

How to work on your local branch

Follow this workflow as you work:

  1. Review the appropriate style guide.
    If you are changing code, review the coding style guide. Changing documentation? Review the documentation style guide.
  2. Make changes in your feature branch.
  3. Test your changes as you work.
  4. For code changes, add unit tests if appropriate.
  5. Format your source files correctly.
  6. List your changes

The status command lists what changed in the repository. Make sure you see the changes you except.

  1. Add your change to git

  2. Commit your changes making sure you use the -s flag to sign your work.

  3. Push your change to your repository.

Pull and rebase frequently

  1. Return to the terminal on your local machine and checkout your feature branch in your local docker-fork repository.

  2. Fetch any last minute changes from docker/docker.

  3. Start an interactive rebase.

  4. Rebase opens an editor with a list of commits.

  5. Replace the pick keyword with squash on all but the first commit.

  6. Edit and save your commit message. Make sure your message includes your signature.

  7. Force push any changes to your fork on GitHub.
