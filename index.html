编译选项的含义[host, build, target]

Configure with —host, —target and —build options

When people configure a project like ‘./configure’, man often meets these three confusing options, which are more related with cross-compilation

  • —host: In which system the generated program will run.
  • —build: In which system the program will be built.
  • —target: this option is only used to build a cross-compiling toolchain. When the tool chain generates executable program, in which target system the program will run.

An example of tslib (a mouse driver library)

'./configure --host=arm-linux --build=i686-pc-linux-gnu': 
the dynamically library is built on a x86 linux computer but will be used for a embedded arm linux system.

An example of gcc:

'./configure --target=i686-pc-linux-gnu --host=arm-linux --build=i686-pc-linux-gnu':
A gcc compiler is build on x86 linux computer. [--build]
The gcc will be executed on an embedded arm linux system.[--host]
But the gcc will generate the binary program, which run in a x86 linux system. [--target]

ios远程调试

安装debugserver

ios默认是无法通过lldb调试的，默认只能通过xcode连接调试，有时候很不方便，能不能通过lldb远程连接调试呢？答案是可以的。

下面以ios8.4为例，详细说明具体配置过程:

  1. 首先iPhone需要越狱，越狱成功后安装OpenSSH服务。

  2. 安装ios8.4 SDK中的debugserver到iPhone上。

    hdiutil attach /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/DeviceSupport/8.4\ \(12H141\)/DeveloperDiskImage.dmg

cd /Volumes/DeveloperDiskImage

cp /usr/bin/debugserver .

  1. 编写entitlements.plist配置文件。

    <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/ PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>com.apple.springboard.debugapplications</key>
    <true/>
    <key>run-unsigned-code</key>
    <true/>
    <key>get-task-allow</key>
    <true/>
    <key>task_for_pid-allow</key>
    <true/>
</dict> 
</plist>

  2. 使用codesign签名。

    codesign -s - -- entitlements entitlements.plist -f debugserver

  3. 拷贝已签名的debugserver到iphone上。

    scp /usr/bin/debugserver root@192.168.1.2:/usr/bin/

调试程序

  1. 启动debugserver调试目标程序, debugserver支持直接启动程序和附加到程序进程进行调试，可以直接输入debugserver查看参数帮助。

    export $TARGET=/path/to/target/on/iphone

ssh root@192.168.1.2 debugserver *:1234 $TARGET

  2. 启动lldb连接到远程debugserver开启的端口，进行调试。

    lldb

(lldb): platform select remote-ios

(lldb): process connect connect://192.168.1.2:1234

  3. 通过wifi连接调试很慢，也可以通过USB进行调试。

    • 安装usbmuxd。可以用来通过usb端口建立iphone的代理连接。
    brew install usbmuxd
    • 把iphone线接到电脑上，然后启动代理连接。
    iproxy 1234 1234
    • 现在我们可以直接连接到本地1234端口来进行调试。
    lldb

(lldb): platform select remote-ios

(lldb): process connect connect://localhost:1234

好了，进行享受lldb远程调试吧！

  1. iphonedevwiki-debugserver

Mac osx常见反调试手段

在软件发布后，你肯定不情愿你刚刚发布的软件就出现XXX破解版，而延长软件破解很重要的手段之一－－－阻止破解者对你的软件进行调试。
以下有几种常见的反调试手段:

1. 使用ptrace函数禁止附加调试, ptrace函数的声明是:

int ptrace(int request, pid_t pid, caddr_t addr, int data);

参数request的值为PT_DENY_ATTACH时，即可禁止附加调试。


    #include <sys/types.h>
    #include <sys/ptrace.h>
    #include <stdio.h>

    int main(int argc, char *args[]) {
        ptrace(PT_DENY_ATTACH，0，0，0)；
        printf("Hello world\n");
        return 0;
    }

mac应用程序切换默认语言设置

最近Mac上用Excel比较多，所有安装了最新Office for mac 2016, 由于系统语言是英文版的，所以打开Excel时，默认也是英文环境，用起来不是很方便，所以想能不能在修改系统语言设置下，让Excel启动在中文环境下。

Mac应用程序和语言环境变量有以下三个。

  • AppleLanaugages 控制语言列表
  • AppleLocale 控制语言相关地区设置，包括日期、货币等相关格式
  • AppleCollationOrder 控制列表排序

所以我们可以通过改变AppleLanaugages来设置App语言，下面是两种常见方法。

  1. 启动Terminal, 使用defaults来改变软件语言设置。

    defaults write com.microsoft.Excel AppleLanguages -string "(zh-CN)" //修改设置

defaults delete com.microsoft.Excel AppleLanguages //删除设置

  2. 通过软件来修改软件,如App Language Chooser、languageswitcher等等。

色彩理论(wiki摘要)

色彩概述

20世纪初色彩理论原则围绕着”纯的”或者说”理想”色彩而建立, 是用来描述感觉体验而非物理世界中色彩的属性。这导致原有的色彩理论原则并不能很好的适应现代的构想。

主要有下面3个问题:

  1. 色光混合(加色法)和色料混合(减色法)之间行为的困惑。这个困惑在于材料对光的吸收和眼睛对光的感知遵循不同的规则。

  2. 在描述从一个表面(如油漆或颜料)反射出来的色彩产生强烈亮度对比度这个重要效果时, 色料混合和色光混合是截然相反的; 色彩如棕色(browns)或赭色(ochres)无法出现在色光混合中。因此，中间值的黄色油漆和周围的亮白色之间的强烈亮度对比使得黄色呈现出绿色或棕色，而彩虹和周围的天空之间的强烈亮度对比使得黄色在彩虹中出现微弱的黄色或白色。

  3. 精确的描述色彩效果的需要。当大多数色彩效果都能以三个相对属性来定义反差时，就能以如”黄色”和”蓝色”之间反差构思出更通用的色彩。

    • 亮度(lightness)[light vs. dark, or white vs. black]
    • 饱和度(saturation)[intense vs. dull)]
    • 色相(hue)(如red, orange, yellow, green, blue or purple)

这些问题一直到19世纪末也没有解决。

Git将一个分支的提交合并到其他分支

Git merge dev to release-N

问题:

你有4个不同分支，dev, release-1, release-2, release-3, release-4, 你想把dev分支上所做的提交合并到release-x分支上，以便所有release-x分支都能获得最新的dev提交.

方案一: checkout每个目标分支，然后并入dev分支提交.

git checkout release-1
git merge dev

git checkout release-2
git merge dev

...
git checkout release-N
git merge dev
